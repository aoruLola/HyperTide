# Security Policy

HyperTide is currently maintained as a private commercial codebase.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately to the maintainers instead of opening a public issue or pull request.

Include:

- affected component or endpoint
- reproduction steps
- impact assessment if known
- logs, screenshots, or request samples when relevant

## Response Expectations

We will review reports privately, validate impact, and coordinate remediation before any broader disclosure.

## Do Not

- post exploit details in public issues
- disclose secrets, credentials, or customer data in reports
- attempt disruptive testing against shared environments without approval
